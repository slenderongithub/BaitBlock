# Developer Onboarding

## Fastest Path to a Running App (recommended: Node backend)

```bash
cd /Users/slender/Developer/Codes/ClickbaitDetection
npm install
npm start
```

Open `http://localhost:3000`, paste any public news article URL, click Analyze. If port 3000 is busy: `lsof -ti tcp:3000 | xargs kill -9`.

That's the entire live app — no database, no build step, no env file required.

## If You Need the Python (NLP) Backend

It is **not currently runnable as shipped** — see [[Known-Issues]] #1 for the broken static path and #11 for the undocumented spaCy model download. To get it working:

```bash
python -m venv .venv
source .venv/bin/activate   # or .venv\Scripts\activate on Windows
pip install -r requirements.txt
python -m spacy download en_core_web_sm
# Fix app.py's static_folder to point at ../public before this will actually serve the UI (see Known-Issues #1)
python app.py
```

Expect the first run to download the `all-MiniLM-L6-v2` sentence-transformer model unless you've pre-populated `.cache/huggingface`.

## Things That Will Surprise You

- `script.js` and `server.js` at the repo root are **not** the files that run — the real ones are `public/script.js` and `src/server.js`. See [[Known-Issues]] #3.
- The "NLP Metrics" panel in the UI (cosine similarity, sentiment polarity) means something different depending on which backend served the response — see [[Glossary]] and [[Known-Issues]] #2.
- The evaluation charts in `README.md` are not generated by running this app against real articles — see [[Known-Issues]] #6.
- There are no tests to run, no linter to satisfy, and no CI to wait on — verification today is manual (`npm start` + try a few URLs).
